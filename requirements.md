# 斗鱼鱼吧自动签到插件需求文档

## 🎯 需求总览
开发一个 Chrome 扩展插件，实现斗鱼鱼吧自动签到、主播管理等功能，提升鱼吧签到效率。

---

## ✅ 已实现需求

### 需求1：鱼吧页面自动签到
**触发条件**：访问 `https://yuba.douyu.com/discussion/{数字}/posts` 格式的网址时
**功能实现**：
- 自动注入脚本，识别页面中文字内容为「签到」的按钮并自动点击
- 签到成功在控制台打印 `✅ 签到成功`，未找到签到按钮打印 `ℹ️ 无需签到，未找到签到按钮`
- 检测 URL 中存在 `open_type=auto_check_in` 参数时，签到完成后3秒自动关闭页面

### 需求2：批量自动签到
**触发条件**：点击 popup 页面的「鱼吧自动签到」按钮
**功能实现**：
- 读取所有已保存的主播列表
- 自动在后台新标签页打开每个主播的签到页面，携带 `open_type=auto_check_in` 参数
- 没有保存主播时弹出提示「你还没有保存任何主播哦~」
- 打开所有页面后自动关闭 popup 窗口

### 需求3：鱼吧列表页增强
**触发条件**：访问 `https://yuba.douyu.com/mygroups` 和 `https://yuba.douyu.com/groupcates/discovery/rec` 页面时
**功能实现**：
- 注入独立脚本 `js/mygroups.js`
- 识别所有鱼吧卡片，提取 href、头像 src、主播名称 title 属性
- 每个主播卡片名称后添加操作按钮：
  - 未保存：显示绿色「添加」按钮，点击保存到本地存储
  - 已保存：显示红色「移出」按钮，点击从本地存储移除
- 支持页面动态变化监听：DOM变化、路由跳转、滚动加载时自动重新处理卡片，按钮永不消失
- 数据保存在 `chrome.storage.local`，以 href 作为唯一标识
- 插件更新导致上下文失效时友好提示用户刷新页面

### 需求4：已保存主播管理
**触发条件**：打开 popup 页面时
**功能实现**：
- 从本地存储读取已保存的主播列表
- 列表项展示主播头像、名称
- 支持排序：按保存时间倒序排列，最近添加/置顶的显示在最前面
- 每个列表项右侧有两个操作按钮：
  - ⬆️ 置顶按钮：点击更新保存时间为当前，自动排到列表最前面
  - ❌ 删除按钮：点击弹出确认弹窗，确认后移除该主播
- 点击主播头像/名称，打开对应鱼吧页面（不带自动签到参数）
- 没有保存主播时显示「暂无主播」提示
- 列表最大高度380px，超出部分可滚动查看

### 需求5：快捷导航
**触发条件**：点击 popup 页面导航按钮
**功能实现**：
- 「我关注的鱼吧」按钮：点击打开 `https://yuba.douyu.com/mygroups`
- 「全部鱼吧」按钮：点击打开 `https://yuba.douyu.com/groupcates/discovery/rec`
- 两个按钮并排显示，各占50%宽度
- 点击后新标签页打开链接，自动关闭 popup


### 需求6：自动签到开关
**触发条件**：popup 页面加载时
**功能实现**：
- 在「鱼吧自动签到」按钮下方添加单独的开关行
- 开关左侧显示说明文字：「每天第一次访问斗鱼时自动签到」
- 开关默认状态为关闭
- 开关状态变化时自动保存到 `chrome.storage.local` 持久化存储

### 需求7：全局自动签到触发
**触发条件**：访问任意 `*.douyu.com` 域名下的页面时
**功能实现**：
- 新增独立注入脚本 `js/auto-sign.js`，匹配所有斗鱼站点（包括主站、鱼吧等所有子域名）
- 脚本加载时自动读取「自动签到」开关状态
- 开关开启时，检查本地存储中今日是否已执行过自动签到
- 今日未执行的情况下，自动触发批量自动签到流程，后台打开所有已保存主播的签到页面
- 签到执行完成后，将今日日期持久化存储到 `chrome.storage.local`
- 同一天内不会重复触发自动签到流程，避免重复打开页面

### 需求8：关注列表自动签到开关
**触发条件**：popup 页面加载时
**功能实现**：
- 在「每天第一次访问斗鱼时自动签到」开关的下方，添加一个新的开关
- 开关左侧显示说明文字：「直接签到我关注的鱼吧（无需手动添加主播）」
- 开关默认状态为关闭
- 开关状态变化时自动保存到 `chrome.storage.local` 持久化存储
- 开关打开时的影响：
  1. popup 页面不显示已保存主播列表
  2. 在 mygroups.js 中不需要显示添加和移出按钮
  3. 执行鱼吧自动签到时，直接打开 `https://yuba.douyu.com/mygroups?open_type=auto_check_in` 即可，无需遍历打开单个主播页面

### 需求9：mygroups页面自动滚动批量签到
**触发条件**：访问 `https://yuba.douyu.com/mygroups?open_type=auto_check_in` 页面时
**功能实现**：
- 新增独立注入脚本 `js/mygroups-auto-sign.js`，仅注入到mygroups页面
- 页面加载时自动检测URL中是否存在 `open_type=auto_check_in` 参数，不存在则不执行
- 自动滚动页面到底部，直到出现「没有更多内容了」提示，加载完所有关注的鱼吧
- 收集页面上所有鱼吧的链接信息，自动去重
- 通过 `sendMessage` 调用现有的 `autoSign` 接口，将收集到的鱼吧数组发送到background.js
- background.js自动打开所有鱼吧的签到页面，完成批量签到
- 所有请求发送完成后，自动关闭当前mygroups页面
